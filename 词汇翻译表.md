## 4. 词法细节

### 4.1 标识符 `IDENT`

```
[_A-Za-z][_A-Za-z0-9]*
```

- 符合 C 标识符规范。
- 匹配后查关键字表；命中则返回对应 `KW_*`，否则返回 `IDENT`。

### 4.2 整数 `NUMBER`

```
NUMBER ::= 0 | [1-9][0-9]*
```

- 仅匹配**非负**整数字面量。
- `Token` 保存原始数字拼写（raw lexeme），不做符号处理。
- `-` 始终作为独立的 `MINUS` Token 输出，负数的语义由 **Parser 在 `UnaryExpr`** 中通过一元取负处理。
- 词法层无需负号消歧逻辑。

示例：

| 源码片段      | Token 序列                                           |
| ------------- | ---------------------------------------------------- |
| `return -5;`  | `KW_RETURN` `MINUS` `NUMBER(5)` `SEMICOLON`         |
| `a - 5`       | `IDENT(a)` `MINUS` `NUMBER(5)`                      |
| `a-5`         | `IDENT(a)` `MINUS` `NUMBER(5)`                      |
| `(-1)`        | `LPAREN` `MINUS` `NUMBER(1)` `RPAREN`               |

### 4.3 数字幅值与边界处理

- 常量求值（semantic analysis 阶段）以**至少 64 位有符号中间类型**（`int64_t`）处理数字幅值，避免中间计算溢出。
- 最终结果截断为 32 位有符号 `int`（`int32_t`）写入 IR / 生成代码。
- **`-2147483648` 边界方案**：源码中的 `-2147483648` 词法序列为 `MINUS NUMBER(2147483648)`。Parser 构造 `UnaryExpr(-, 2147483648)`。语义分析以 `int64_t` 计算 `-2147483648`，结果为合法的 `int32_t` 最小值，截断后正确。

### 4.4 空白与注释（跳过，不产生 Token）

| 类型       | 规则                                       |
| ---------- | ------------------------------------------ |
| 空白       | 空格、`\t`、`\r`、`\n`                     |
| 单行注释   | `//` … 到行末（不含换行）                  |
| 多行注释   | `/*` … `*/`（支持跨行；未闭合则词法错误）  |

### 4.5 双字符运算符优先级

扫描时优先尝试双字符，再回退单字符：

```
||  &&  <=  >=  ==  !=
```
